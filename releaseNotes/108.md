## 1.0.9

### 新功能

- **CLI 自动检测**: 插件现在会自动检测多个常见安装路径下的 OpenCode CLI，包括：
  - `~/.opencode/bin/opencode` (npm 全局安装)
  - `/opt/homebrew/bin/opencode` (macOS ARM Homebrew)
  - `/usr/local/bin/opencode` (macOS Intel Homebrew / Linux)
  - `/home/linuxbrew/.linuxbrew/bin/opencode` (Linux Homebrew)
  - 以及其他常见路径

### 修复与优化

- **远程连接修复** (Issue #14): 修复了连接到远程 OpenCode 服务器后，点击 "Restore UI" 时崩溃的问题。现在远程连接模式下会正确恢复连接状态，而不是尝试创建本地终端。
- **Snap/沙箱环境兼容性**: 修复了在 Snap 版 PyCharm 等沙箱环境中无法找到 Homebrew 安装的 OpenCode CLI 的问题。插件现在会直接检查文件路径，不再依赖系统 PATH 环境变量。
- **Linux/Mac 兼容性**: 用户不再需要手动创建 `/usr/local/bin/opencode` 软链接。
- **Windows 连接修复**: 修复了 Windows 平台下因 OpenCode 服务启动较慢导致的连接超时问题。
- **增强连接稳定性**:
  - 将服务启动等待时间延长至 30 秒。
  - 即使初始连接超时，也会在后台持续尝试重连。
  - 增加了 HTTP 请求的超时宽限。
