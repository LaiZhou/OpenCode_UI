# OpenCode Plugin 1.0.2 Release Notes

本次更新集中修复了中文支持问题、Terminal 交互体验，并全面优化了 Diff Code Review 流程，对标 Claude Code 的原生体验。

## 🚀 新特性与体验优化

- **Diff 流程升级**
  - **每轮独立展示**：新消息产生的 Diff 独立展示，自动清除旧的未处理 Diff（视为隐式接受），保持工作区整洁。
  - **自动导航**：点击 Accept/Reject 后自动关闭当前 Diff 面板，并流畅跳转至下一个待处理文件。
  - **生成顺序 Review**：Diff 列表改为按 AI 生成时间升序排列，符合思维链（Thought Process）阅读习惯。
  - **界面简化**：移除 Diff 标题中令人困惑的动态计数器（如 `1 of 5`），仅展示文件名，减少认知负担。

## 🐛 Bug 修复

- **中文文件名支持**
  - 修复了新建中文文件名文件时，Diff 面板内容为空的问题。
  - 修复了服务端返回文件名包含八进制转义导致乱码的问题。
  - 增加了磁盘读取回退机制，确保在服务端数据异常时仍能正确显示文件内容。

- **Terminal 交互修复**
  - 修复了点击"添加文件"粘贴到 Terminal 时，光标乱跳或插入位置错乱的问题。
  - 优化为原子写入操作，提升了命令输入的稳定性。
  - 修复了 Untracked 文件在 Diff 列表中重复显示的问题，实现了更精准的增量变更检测。

## 🛠️ 其他改进

- 优化了 Diff 数据反序列化逻辑，增强了对特殊字符的容错能力。
